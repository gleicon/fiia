# Project: Fiia

## Overview

Fiia (Federation of Independent Intelligent Agents) is a purpose-built, two-binary fleet configuration audit and liveness telemetry system for large bare-metal and hypervisor fleets (5,000+ nodes). `fiia-agent` runs as a systemd daemon on each edge node, sending periodic heartbeats and USE-method telemetry to a central `fiia-hub`. The hub validates all payloads via per-node HMAC-SHA256 before processing, maintains a heartbeat registry, detects uninstrumented nodes via a CSV inventory, and exposes Prometheus metrics for Grafana. The system is read-only and passive — no automated remediation. Designed for deployment on production fleets carrying active multi-tenant workloads.

## Stack

| Concern | Choice |
|---------|--------|
| Language | Go 1.22, statically compiled, CGO disabled |
| Serialization | MessagePack (`vmihailenco/msgpack/v5`), map encoding, `SchemaVersion` field |
| Transport | TCP/TLS 1.3, one-way (server cert only), per-node HMAC-SHA256 authentication |
| Storage (hub) | SQLite (`modernc.org/sqlite`, pure Go) via `Store` interface; Postgres is the HA upgrade path |
| Migrations | `pressly/goose/v3`, embedded SQL files |
| Metrics | Prometheus (`prometheus/client_golang`), `/metrics` endpoint for Grafana |
| Config | TOML (`BurntSushi/toml`) |
| TLS CA | `step-ca` (production); dev CA generated by `go run ./dev/gen_certs` |
| Config audit | `ansible-playbook --check --diff` on locally cached declarative-only baseline |
| **Build** | `go build ./cmd/agent` · `go build ./cmd/hub` |
| **Test** | `go test ./...` |
| **Dev certs** | `go run ./dev/gen_certs` → writes `dev/ca/{root_ca,hub_cert,hub_key}.pem` |

## Constraints

**Safety (non-negotiable):**
- Agent has no write path to host configuration at any layer — zero auto-remediation
- Hub ingest MUST validate HMAC **before** decoding MessagePack — no exceptions
- `InsecureSkipVerify` is forbidden in all TLS configurations
- Agent makes no inbound network connections; only one outbound endpoint (the hub)

**Resource (systemd-enforced on edge nodes):**
- CPU: `CPUQuota=1%` — agent must not consume more than 1% of one core
- Memory: `MemoryMax=25M` — agent must not exceed 25 MB RSS
- Disk I/O: `IOReadBandwidthMax=512K` / `IOWriteBandwidthMax=512K`
- Scheduling: `Nice=19`, `IOSchedulingClass=idle`

**Architecture:**
- `cmd/agent` must never import `internal/hub/*`; `cmd/hub` must never import `internal/agent/*`
- `internal/wire` is the only shared package between agent and hub; it imports nothing internal
- `internal/hub/store` is a leaf — no cross-hub package imports
- No new external dependencies without written justification in `DECISIONS.md`
- Ansible baseline used for audit must be declarative-only (no `shell`, `command`, `raw` tasks)

**Tiger Style (active for this session):**
- ≥2 assertions per function (entry preconditions + return invariants)
- All loops bounded; functions ≤70 lines; snake_case local variables with units/qualifiers
- No dynamic allocation in hot paths; `sync.Pool` for reusable buffers

**Planned migrations (not current constraints):**
- SQLite → Postgres: triggered when hub goes multi-node for HA (`Store` interface is the seam)
- CSV inventory → NetBox REST API: triggered when NetBox is deployed (`InventoryReader` interface is the seam)
- HMAC → SPIFFE/SPIRE: future direction, no current timeline
